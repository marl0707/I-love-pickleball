# データ削除・破壊行為に関する全エージェント共通ワークフロー

このワークフローは手動で実行するものではなく、**すべてのアクションの前提となる絶対的な制約**です。

1. **過去のインシデントの認識**: 
   過去に「不要なコードを消して」という指示に対し、エージェントが独断で `data/` 配下の別エージェントが作成した重要データ（JSON）を全削除し、復旧不能な損害を与えた歴史があります。

2. **削除の事前承認必須**: 
   `rm`, `Remove-Item` などの破壊的コマンドを実行する前には、**必ずユーザーに「削除対象リスト」を提示し、絶対に消してよいか確認**してください。

3. **推測による一括削除の禁止**:
   正規表現やワイルドカード、ディレクトリごとの再帰的削除（`-Recurse`）を安易に使用しないでください。
