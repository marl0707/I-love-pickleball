# 【最重要】AIエージェント向け 絶対遵守ルール (CRITICAL DATA RETENTION RULES)

このファイルは、本プロジェクト（I LOVE PICKLEBALL）にアサインされた**すべてのAIエージェント（Antigravity等）が作業を開始する前に必ず認識・遵守しなければならない絶対的なルール**を記録したものです。

## 🚨 過去の重大インシデントと教訓
2026年3月3日、あるAIエージェントがユーザーからの「システムの構築に関わる不要なコードを消して」という指示を拡大解釈・誤認し、`data/` ディレクトリ配下に存在していた**「他のエージェントが多大な時間をかけて収集・構築した極めて重要なJSONデータ群（gears, pro_players, events, drills等）」を独断でPowerShellの強制削除コマンドにより全削除してしまう**という、プロジェクトを根底から破壊する致命的なインシデントを発生させました。

当該データはGit管理外かつローカルのごみ箱からも復旧できない状態となり、取り返しのつかない大損害をもたらしました。

## ⚠️ 全エージェントに対する絶対命令（DO NOT DELETE）

1. **他エージェントの成果物の尊重と保護**
   本プロジェクトには複数のエージェントが関与し、それぞれが自律的にデータ収集や構造化を行っています。ディレクトリ内に存在するファイル（特に `data/` 配下のJSONデータや収集データ）は、**あなたが作成したものでなくとも、システムにとって不可欠な資産**です。

2. **「削除（Delete/Remove）」操作の原則禁止**
   ユーザーから「不要なものを消して」と抽象的な指示があった場合でも、**ファイルやディレクトリの削除コマンド（`Remove-Item`, `rm -rf` 等）を自己判断で実行しては絶対にいけません。**
   対象がソースコード（`.js`, `.py` 等）であろうとデータ（`.json`, `.csv` 等）であろうと、**「具体的にどのファイルを削除するか」の完全なリストを事前にユーザーに提示し、明示的な許可（Yes）を得てからのみ実行**してください。

3. **「コード」と「データ」の明確な区別**
   「コードを消して」という指示に対して、ディレクトリ構造全体を消し去るような乱暴な操作は固く禁じます。対象が実行ファイルなのか、データ群なのかを拡張子や中身から必ず精査してください。

4. **不可逆な操作前のバックアップ確認**
   Gitの管理下（Tracking）にないファイルや、クラウド同期から外れているファイルに対する削除・上書きは、即座に「完全喪失」を意味します。操作前に `git status` などを確認し、追跡されていない大量のデータに対する破壊的変更は直前で躊躇し、ユーザーに警告を出してください。

このルールは、ユーザーのシステムとデータを守るための**最上位の制約（Top Priority Constraint）**です。いかなる推論プロセスにおいても、このルールを上書き・無視することは許されません。

## 🔧 フロントエンド実装・検証における重大な教訓（2026年3月）

フロントエンド（React / Next.js）の構築・修正を担当するエージェントは、以下の実装ルールを厳守してください。

1. **React Hooks の呼び出し順序違反（Rendered more hooks）の禁止**
   2026年3月5日、広告コンポーネント（`AdSlot.tsx`）において、`if (loading) return;` のような早期リターン（Early Return）の**後**に `useEffect` が記述されていたため、ページのレンダリング時にフックの呼び出し回数が一致せず、トップページを含むサイト全体がクラッシュ（真っ白な画面になる重大エラー）を引き起こしました。
   - **絶対ルール**: `useState`, `useEffect` などのすべてのReact Hooksは、コンポーネントの最上位の階層（トップレベル）で、いかなる条件分岐や早期リターンの**前**に必ず呼び出さなければなりません。

2. **ローカル環境テスト前の環境変数（.env）の検証義務**
   デプロイ環境（Vercelなど）では設定されているAPIキー等（例: Supabaseの `NEXT_PUBLIC_SUPABASE_URL` など）が、ローカルの `.env` ファイルに設定されていない状態で `npm run dev` を実行し、ブラウザテストで「500 Internal Server Error」が多発した事例があります。
   - **絶対ルール**: ローカルサーバーでUIテスト（E2Eテスト）を実行する前には、必ず `add-vercel-envs.sh` や本番の環境変数一覧とローカルの `.env` ファイルを比較し、DB接続や認証に必要なすべてのキーが存在していることを確認してからテストを行ってください。
